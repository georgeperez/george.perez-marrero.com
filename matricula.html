RUMAD on macOS Sierra and up (including High Sierra)

RUMAD en macOS Sierra en adelante (incluyendo High Sierra)

START WARNING

It's important that you read these instructions from your desktop and to do so diligently. If the instructions below (after this warning) don't work for you, you're going to need to enter the following command each time you want to connect to rumad.uprm.edu in Terminal:

ssh -oKexAlgorithms=+diffie-hellman-group1-sha1 -oHostKeyAlgorithms=+ssh-dss estudiante@rumad.uprm.edu

Press Return.

Using the aforementioned command, your Mac should connect to rumad.uprm.edu. If you need any help, my email is at the end of this page.

I'm not responsible for any damages done to your computer from following these instructions.

Manually copy and paste the commands.

Your Terminal window might look differently from mine. These instructions will still work no matter how yours looks.

END WARNING

First step

NOTE

The $ symbol indicates the command prompt of your Terminal. Everything that is after the $ symbol is a command. DO NOT include it in your commands.

Open a new Terminal window. In Terminal, make sure you're in the correct directory. Type the following command: 

$ pwd
Press Return. The following should appear:

$ pwd
/Users/your-username

On your computer, your-username will be your username.

If the command pwd showed /Users/your-username it means that you're in the correct directory. If it didn't, you'll need to change directories. Type the following command:

$ cd

Press Return. Run the pwd command once again and confirm that you're in the right directory.

Once you've confirmed the directory, type the following command: 

$ open .ssh/config

Press Return. Running this command should've opened a TextEdit window.

Possible problems and their fixes

1. "The file /Users/your-username/.ssh/config does not exist"

If your Terminal shows the following error: 

The file /Users/your-username/.ssh/config does not exist
It means that the config file doesn't exist on your Mac. To fix this, type the following command to create the file: 
$ touch .ssh/config

Press Return. Afterwards, type the previous command: 

$ open .ssh/config
Press Return once more.

2. "No such file or directory"

It's possible that the .ssh/ directory does not exist on your Mac. Your Terminal window will show the following error: 

.ssh/: No such file or directory

In this case, you'll need to create it. Type the following command: 

$ sudo mkdir .ssh

NOTE

The sudo part of the previous command elevates your Terminal command prompt to use admin privileges.

The command prompt will ask for your user password.

Password:

Type your user password. The password will not appear on-screen, but rest assured it is being written. When you finish typing your password, press Return.

Afterwards write these commands in order:

$ touch .ssh/config

Press Return.

$ open .ssh/config

Press Return one more time.

Second step

Your Mac must've opened a new TextEdit window with the document config. This file will be empty for you.

In the config file, copy and paste the following text:

Host rumad.uprm.edu
	  KexAlgorithms +diffie-hellman-group1-sha1
	  HostKeyAlgorithms +ssh-dss

Your config file should look like the screenshot below. Make sure it does before continuing.

Third step

Save your changes using the Command+S keyboard shortcut or by navegating to File -> Save in TextEdit.

Connect to rumad.uprm.edu using the following command: 

$ ssh estudiante@rumad.uprm.edu

Press Return. If everything is working correctly, you should see a password prompt for the user estudiante. Since estudiante does not have a password, press Return again.

Fourth step

SUCCESS! If everything was done correctly, your Terminal will show the user menu for the Sistema Estudiantil Colegial.

Please send any questions or feedback to my email.
