RUMAD on macOS Sierra and up (including High Sierra)

RUMAD en macOS Sierra en adelante (incluyendo High Sierra)

It's important that you read these instructions from your desktop and to do so diligently. If the instructions below don't work for you, you're going to need to enter the following command every time you want to connect to rumad.uprm.edu in Terminal:

ssh -oKexAlgorithms=+diffie-hellman-group1-sha1 -oHostKeyAlgorithms=+ssh-dss estudiante@rumad.uprm.edu

Press Return.

Using the aforementioned command, your Mac should connect to rumad.uprm.edu. If you need any help, my email is at the end of this page. I'm not responsible for any damages done to your computer from following these instructions. Manually copy and paste the commands.

Finally, your Terminal window might look differently from mine. These instructions will still work no matter how yours looks.

Set up the SSH directory

NOTE

The $ symbol indicates the command prompt of your Terminal. Everything that is after the $ symbol is a command. DO NOT include it in your commands.

Open a new Terminal window. Terminal is located in /Applications/Utilities/Terminal.app.

Type the following command to create the SSH directory: 

$ mkdir ~/.ssh

Press Return.

Then, change the permissions of the folder to be accessible by you only. Type the following command: 

$ chmod 700 ~/.ssh

Press Return.

Afterwards write this command:

$ nano ~/.ssh/config

Press Return.

Whitelist rumad.uprm.edu

Your Terminal window will show a document. This document will likely be empty for you.

In the config file, copy and paste the following text using the Command+V keyboard shortcut:

Host rumad.uprm.edu
	  KexAlgorithms +diffie-hellman-group1-sha1
	  HostKeyAlgorithms +ssh-dss

Your config file should look like the screenshot below.

Save your changes using the Control+S keyboard shortcut, then close the text editor by using Control+X.

Testing out the connection

Connect to rumad.uprm.edu using the following command: 

$ ssh estudiante@rumad.uprm.edu

Press Return. If everything is working correctly, you should see a password prompt for the user estudiante. Since estudiante does not have a password, press Return again.

Eureka!

If everything was done correctly, your Terminal will show the user menu for the Sistema Estudiantil Colegial.

Configuración del directorio SSH

OJO

El símbolo $ indica el prompt de comando en tu Mac. Todo lo que está después del símbolo $ es un comando para entrar (copy/paste). NO lo incluyas en tus comandos.

Abre una nueva ventana de Terminal. Este está localizado en /Applications/Utilities/Terminal.app.

Escribe el siguiente comando para crear el directorio de SSH: 

$ mkdir ~/.ssh

Después, cambia los permisos del directorio para que solo sea accesible por ti: 

$ chmod 700 ~/.ssh

Oprime Return.

Finalmente, escribe el siguiente comando: 

$ nano ~/.ssh/config

Oprime Return una vez más.

"Whitelisting" a rumad.uprm.edu

Tu Terminal debe haber abierto un nuevo documento. Para tí, este documento lo más probable está vacío.

Este documento se llama el config file. Dele copy al siguiente bloque de texto:

Host rumad.uprm.edu
	  KexAlgorithms +diffie-hellman-group1-sha1
	  HostKeyAlgorithms +ssh-dss

Vaya a la ventana de Terminal que contiene el documento config. Dele paste con la combinación de teclado Command+V.

Tu config debe lucir como el siguiente screenshot.

Guarda tus cambios usando la combinación de teclado Control+S y sal del editor de texto usando Control+X.

Probando la conexión

Conéctate a rumad.uprm.edu usando el siguiente comando: 

$ ssh estudiante@rumad.uprm.edu

Oprime Return. Si todo está funcionando bien, deberias ver el prompt para entrar el password de estudiante.

Oprime Return ya que estudiante no tiene password.
