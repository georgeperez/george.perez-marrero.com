RUMAD on macOS

As a consequence of a recent upgrade by the Centro de Tecnolog√≠as de Información (CTI), it is possible that when trying to reconnect to RUMAD (Matricula, colloquially/erronously known as "PuTTY") after November 11, 2019 you will see the following warning message:

[~] $ ssh estudiante@rumad.uprm.edu
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@     WARNING: REMOTE HOST IDENFITIFICATION HAS CHANGED!    @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY!
Someone could be eavesdropping on you right now (man-in-the-middle-attack)!
It is also possible that a host key has just changed.
The fingerprint for the RSA key sent by the remote host is
[...]
Please contact your system administrator.
Add correct host key in Users/[...]/.ssh/known_hosts:1
RSA host key for [...] has changed and you have requested strict checking.
Host key verification failed.
[~] $

To fix this, simply copy and paste the following command in Terminal:

ssh-keygen -R rumad.uprm.edu

Press Return. This will regenerate the RUMAD key.

Once this is done, reconnect to rumad.uprm.edu using the following command (copy and paste into your Terminal window):

ssh estudiante@rumad.uprm.edu

Press Return. You will connect to the RUMAD system successfully.
